## USBがどちらに刺さっているかの識別。
### 背景
- [SPEXEL+SPRESENSEの写真]`https://device.risner.jp/html/upload/save_image/0405171247_642d2d7ff09c0.jpeg`で分かる通り、スペースの都合上、2つのUSBコネクタが干渉する。
  - SPRESENSE基板のUSB:CP2xxx経由のUART
  - SPEXEL基板のUSB:SpritzerのUSBに接続されているUSBコネクタ。
  - 電源供給は両方のVBATの並列接続なので、SPEXEL基板側USBコネクタにケーブルを差しても電源は入る。

### 想定ユースケース
- FT実施時
  - SPRESENSEのUSBコネクタにケーブルを差し、電源供給後、ELTRES動作に入る。
  - シリアルログ出力あり。
    - SPRESENSE基板にPCで接続して、ログ保存可能。
  - SDカードにもログ出力あり。
    - SDカードにログ出力は定期的にclose/openして突然の電源断であっても最悪最後のログ消失に抑える。
    - SPEXEL基板のボタン押下でSDカードへログ出力停止,close。
  - 将来的にはUART2にもログ出力させたい。
    - BLE1507のNUS( Nordic UART Service )経由でスマホとつないでリアルタイムモニタリングできると嬉しい。
- FT終了後
  - SPEXELのUSBコネクタにケーブルを差し、電源供給後、MSC動作に入る。
    - 保存されたログをPCから吸い上げる。
  - この時`/dev/ttyACM0`でnshが動くとデバッグ効率上がる。
- 最終的な物理形態
  - SDカードは入れっぱなし、箱に入れて2つのUSBケーブルの差し込み口で、動作を自動的に切り替えたい。

### 困りごと
- SPEXEL側USBにHOSTが接続していることをSPRESENSE-Appで識別できない。
  - FT終了後のデータ吸い上げ時はSPEXELのボタンを押下したままUSBケーブルを差し込むことで識別させている。
  - 何らしかのアプリケーションを作ることでHOSTの接続有無が判別出来れば、その戻り値でログ取りさせるかMSC動作に入るかを決められる。

- SPEXEL側USBコネクタにケーブルを差しているとき、SPRESENSEのUSBコネクタにケーブルを差せないため、nshが使えない。
  - USB HostがSPEXEL側に繋がっているときのみ、`/dev/ttyACM0`でnshを動かしたい
